With the ever rising complexity of the react ecosystem with their `next.js` and server side rendering complicating both development and deployments, a backlash has been rising in the community. Nextjs puts the focus on it's internal complexity instead of the customer problem you are supposed to solve with it. [The market for lemons](https://infrequently.org/2023/02/the-market-for-lemons/) is a writeup of these issues, something I have also noticed with the vanishment of create react app. So vite has a replacement for it, but there's also a lot of good things said about good old server side templating with new tools like htmx. The good old server templating also solves the SEO issues without complex solutions.
In case SPAs would be needed, then some of the replacements for react are (from the lemon reference above): 
- [Stencil](https://stenciljs.com/)
- [Lit](https://lit.dev/) and [Polymer](https://polymer-library.polymer-project.org/3.0/docs/devguide/feature-overview)
- [Svelte](https://svelte.dev/)
- [Preact](https://preactjs.com/)
- [Solid](https://www.solidjs.com/)
- [Marko](https://markojs.com/)
- [Inferno](https://www.infernojs.org/)
- [Hyper](https://github.com/WebReflection/hyperHTML)
- [FAST](https://www.fast.design/)
- [Vue](https://vuejs.org/)
- [Qwik](https://qwik.builder.io/)
And some for the good old backend are:
- [Eleventy](https://www.11ty.dev/)
- [Astro](https://astro.build/)
- [Enhance](https://enhance.dev/docs/)
- [SvelteKit](https://kit.svelte.dev/)
- [Fresh](https://fresh.deno.dev/)
It would be a good idea to study this a bit more, with at least having an alternative to react available.
